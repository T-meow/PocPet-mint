# 伙伴日程技能 Lv10 与大师成长实施计划

## 目标

- 将学习、料理、园艺、运动技能上限从 Lv5 提升到 Lv10，四项全部满级预计需要 9 至 12 周。
- 让技能等级持续影响日程消耗、时间、经验和金币，并在 Lv10 后通过大师完成次数继续提供长期目标。
- 全技能 Lv3/Lv6 分别解锁次日 4/5 个候选项，并立即提供 5%/15% 的日程金币总加成。
- 每日最多完成 3 项不变，增加选择而不扩大每日任务负担。

## 核心规则

- 单项技能升级需求依次为 `40/60/90/130/180/260/360/480/620`，总计 2220 XP。
- Lv2 体力消耗 -10%；Lv4 饱腹与心情消耗 -10%；Lv5 时间 -5%；Lv7 XP +15%；Lv8 分类金币 +5%；Lv9 时间总减免提升至 -10%。
- Lv10 后日程不再获得技能 XP，改为在领取时获得对应分类大师完成次数 +1。
- 达到 Lv10 的当次领取不增加大师次数；之后在 10/30/60 次分别解锁体力总减免 -15%、分类金币总加成 +10%、大师跨系统被动强化。
- 大师次数超过 60 后继续记录，但数值不再增长。

## 候选板与经济

- 当前日活动板保存 `boardOfferCount` 快照；技能里程碑导致的候选数量变化在次日 5:00 生效。
- 候选数量为 3/4/5；前三项保持原等级规模组合，第 4 项追加标准活动，第 5 项追加短活动。
- 前四项保证四个分类各出现一次，第五项允许确定性重复；每日完成上限固定为 3。
- 全技能 Lv3 日程金币总加成 5%，全技能 Lv6 提升为 15%；对应分类 Lv8 再加 5%，大师 30 次后分类加成提升为 10%。
- 所有加成按加法叠加并在开始时写入奖励快照，最高为当前基础日程奖励的 125%。

## 大师跨系统被动

- 学习：每日愿望金币 +10%，大师 60 次后 +20%。
- 料理：普通食物正面 `item.effect` +10%，大师 60 次后 +15%；生日蛋糕和金苹果固定效果不变。
- 园艺：新种植和新收获周期时间 -5%，大师 60 次后 -8%；已经生成的 `nextReadyAt` 不追溯修改。
- 运动：清醒体力恢复间隔 -8%，大师 60 次后 -12%；睡眠恢复不受影响。

## 数据与兼容

- `PartnerScheduleState` 升级为 schema v3，新增 `boardOfferCount`。
- `PartnerScheduleSkill` 新增 `masterCompletions`；活动与结果快照新增 `grantsMasterCompletion`。
- schema v2 的 Lv5 技能迁移为 Lv5、0 XP、大师次数 0；当前日期保留原候选数量，次日再按技能里程碑扩展。
- v2 进行中活动和待领取结果保持原时间与奖励，默认不授予大师次数。
- 现有伙伴日程成就扩展保持兼容，“全技能满级”目标同步从 Lv5 调整到 Lv10；全技能 Lv6 和四类大师各完成 10 次分别增加新的稀有成就。
- 行程满满与全能伙伴分别提供 10% 和 20% 的完整日程奖励额外一份概率，不新增日程存档字段。

## 验证

- 覆盖 XP 曲线、多级升级、Lv10、大师次数 10/30/60 和 60 后继续计数。
- 覆盖全技能 Lv3/Lv6 金币立即生效、候选数量次日生效、3 项完成上限和跨日重置。
- 覆盖学习愿望、料理普通/特殊食物、园艺新周期和运动清醒/睡眠恢复。
- 覆盖 schema v2 技能、活动、结果和三候选活动板迁移。
- 覆盖两项隐藏成就概率叠加、完整奖励复制，以及大师次数和每日完成数保持单次计数。
- 运行伙伴日程规则脚本、双语 JSON 解析、`git diff --check` 和 `npm run build`；不执行 Browser 截图 QA。
