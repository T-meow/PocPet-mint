# Classic 终局：共同梦想与纪念抽卡计划

## 1. 目标

- 在 Classic 停止大型内容更新前，提供一个能长期消耗金币、收束现有养成系统并拥有明确结局的终局玩法。
- 让宠物等级、伙伴日程技能、大师次数、花园和日常积累在终局阶段继续有价值。
- 加入纯游戏内金币驱动的抽卡功能，提供单抽、十连、保底和可完成的收藏目标。
- 控制新美术和内容生产成本，优先复用现有宠物活动图片、道具图标和 UI 组件。

## 2. 范围与非目标

### 本轮范围

- 新增“共同梦想”终局页面与四项长期工程。
- 新增梦想阶段投入、阶段结算、最终纪念和无限纪念等级。
- 新增金币兑换道具“回忆券”。
- 新增“回忆抽取”，支持单抽和十连抽。
- 新增永久卡池、公开概率、十连保底、累计保底、重复卡回收和定向兑换。
- 新增终局存档 schema、旧存档迁移、中英文文案和核心规则检查。

### 非目标

- 不加入现实货币购买回忆券或抽卡次数。
- 不加入限时卡池、赛季卡池、每日免费抽或登录连续奖励。
- 不加入排行榜、联网校验、交易、玩家间赠送或云端抽卡。
- 不用抽卡结果卡住共同梦想主线进度。
- 不加入自由摆放装饰、冒险地图、实时战斗或新的大型资源生产系统。

## 3. 解锁条件

- 宠物达到 Lv.30。
- 四项伙伴日程技能全部达到 Lv.6。
- 解锁后首页功能区显示“共同梦想”入口。
- 未解锁时可查看条件进度，但不能投入金币、兑换回忆券或抽卡。

解锁条件的目的，是让终局建立在现有养成完成度上，而不是成为新玩家早期的金币陷阱。

## 4. 共同梦想

四项梦想与伙伴日程技能一一对应：

| 分类 | 梦想 | 最终成果 |
|---|---|---|
| 学习 | 伙伴百科 | 完成一本记录共同生活的百科 |
| 料理 | 四季食谱 | 整理一册四季料理记录 |
| 园艺 | 纪念果园 | 建成一座属于伙伴的纪念果园 |
| 运动 | 漫长环行 | 完成一次象征成长的漫长环行 |

### 4.1 阶段结构

每项梦想包含五个固定阶段：

| 阶段 | 技能要求 | 分类日程要求 | 大师要求 | 金币投入 |
|---|---:|---:|---:|---:|
| 1 | Lv.6 | 累计完成 10 次 | 无 | 1C |
| 2 | Lv.7 | 累计完成 25 次 | 无 | 2C |
| 3 | Lv.8 | 累计完成 50 次 | 无 | 4C |
| 4 | Lv.10 | 累计完成 80 次 | 10 次 | 8C |
| 5 | Lv.10 | 累计完成 120 次 | 30 次 | 15C |

`C` 为实现前根据满级玩家稳定日收入确定的固定金币基准。它只用于设计期换算，发布版本保存明确的整数价格，不根据玩家当前财产或等级动态变化。

- 每项梦想总投入约为 `30C`。
- 四项梦想全部完成约需 `120C`。
- 旧玩家可以用既有积蓄加速，但仍需满足技能和生涯次数要求。
- 分类日程要求读取成就计数器中的生涯完成次数，避免重复维护相同统计。

### 4.2 投入规则

- 金币可以分次投入，进度永久保留且不能撤回。
- 提供“投入 10%”“投入 25%”“补足当前阶段”三个按钮。
- 实际投入不能超过当前阶段剩余需求。
- 补足金额超过设定的大额阈值时需要二次确认。
- 技能或次数尚未达标时仍可提前投入金币，但只有全部条件满足后才能完成阶段。
- 完成阶段后展示一段简短成长记录，不增加新的现实时间等待。

### 4.3 阶段奖励

- 阶段奖励以纪念和收藏为主，不返还大量金币。
- 每个阶段给予少量心心、普通道具或回忆券。
- 每项梦想完成后解锁对应称号和一张固定纪念卡，该固定卡不进入随机卡池。
- 四项梦想全部完成后解锁 Classic 最终纪念页、最终称号和完成凭证。
- 最终纪念文案明确表示 Classic 主线完成，但宠物仍可继续照顾和养成。

### 4.4 无限纪念等级

四项梦想全部完成后解锁“共同纪念等级”：

- 第 1 级需要投入 `10C`。
- 后续每级需求为上一级的 1.35 倍，使用四舍五入后的整数金币。
- 只记录纪念等级、生涯累计投入和每 10 级称号。
- 不继续提供金币、属性或生产效率加成，防止终局经济再次膨胀。
- 该系统作为 Classic 停更后的永久金币出口，不设置等级上限。

## 5. 回忆券与金币兑换

### 5.1 道具定义

- 新增道具 `memory_ticket`，中文名“回忆券”。
- 回忆券属于终局专用道具，只用于回忆抽取，不能使用、出售或赠送。
- 回忆券可以来自共同梦想阶段奖励，但主要来源是金币兑换。
- 普通商店不展示回忆券，兑换入口放在共同梦想页面的“回忆抽取”区域。

### 5.2 兑换价格

- 单张回忆券建议定价为 `0.25C`。
- 提供兑换 1 张和兑换 10 张两个操作。
- 兑换 10 张不打折，避免十连成为绕过金币消耗的最优方式。
- 实现前用满级稳定日收入校准，使一次十连约消耗 2 至 3 天的常规收入。
- 回忆券不设持有上限，但所有计数继续使用现有安全整数钳制规则。

## 6. 回忆抽取

### 6.1 抽取方式

- 单抽消耗 1 张回忆券。
- 十连消耗 10 张回忆券，并一次性展示 10 个结果。
- 抽取开始时一次性生成并写入全部结果，再播放展示动画，避免崩溃或关闭页面后重复抽取。
- 抽卡不占用宠物，不锁定照顾、花园、日程或番茄钟。

### 6.2 永久卡池

卡池不轮换、不限时，分为三个奖励层级：

| 层级 | 基础概率 | 奖励内容 |
|---|---:|---|
| 普通 | 72% | 普通食物、护理品、基础肥料和常见礼物 |
| 珍贵 | 25% | 高级食物、稀有礼物、树苗、增益卡等奖励 |
| 纪念 | 3% | 回忆卡册中的随机纪念卡 |

- 卡池不直接返还金币。
- 所有可抽道具必须在奖励定义中配置数量和折算价值。
- 奖励平均金币折算价值控制在抽取消耗的 35% 至 45%，保证它仍是长期金币消耗系统。
- 金苹果、金苹果树苗等顶级道具只能位于珍贵层的低权重位置，不作为十连固定产物。

### 6.3 十连与累计保底

- 十连至少包含 1 个珍贵或纪念结果。
- 如果基础抽取的 10 个结果均为普通，第 10 个结果重抽为珍贵层。
- 从上一次获得纪念卡后开始累计，连续 50 抽未获得纪念卡时，第 50 抽必定获得纪念卡。
- 抽到纪念卡后累计保底归零。
- 单抽和十连共享同一个累计保底计数，不允许通过切换抽取方式规避或重复触发。
- 十连保底不重置纪念卡累计保底，只有实际获得纪念卡才重置。
- UI 必须显示公开概率、当前累计抽数和距离纪念保底的剩余次数。

### 6.4 回忆卡册与重复处理

- 首版卡册建议包含 16 至 20 张纪念卡。
- 卡面复用现有宠物状态和活动图片，通过裁切、背景色和文字排版形成卡片，不要求新绘制完整 CG。
- 卡片只提供收藏、故事文本和完成度，不增加宠物属性。
- 首次抽到纪念卡时永久登记到卡册。
- 重复普通纪念卡转化为 10 个“回忆碎片”。
- 重复稀有纪念卡转化为 25 个回忆碎片。
- 100 个回忆碎片可以在尚未获得的普通纪念卡中任选一张兑换。
- 200 个回忆碎片可以在尚未获得的稀有纪念卡中任选一张兑换。
- 全卡册完成后解锁称号；继续抽到重复卡仍记录生涯重复次数，但不再提供新的数值成长。

回忆碎片只用于处理重复卡和确保卡册最终可完成，不进入商店、梦想工程或其他经济系统。

## 7. 数据结构与确定性

新增独立的 `ClassicEndgameState`，初始 schema 为 v1：

```ts
interface ClassicEndgameState {
  schemaVersion: 1;
  projects: Record<PartnerScheduleCategory, DreamProjectProgress>;
  completedAt?: number;
  legacyLevel: number;
  legacyCoinsInvested: number;
  memoryDraw: MemoryDrawState;
}

interface MemoryDrawState {
  totalPulls: number;
  pullsSinceMemoryCard: number;
  rngSeed: string;
  rngCounter: number;
  ownedCardIds: string[];
  duplicateCountsByCardId: Record<string, number>;
  memoryShards: number;
  recentResults: MemoryDrawResult[];
}
```

- 卡池定义、卡片文本、图片映射和奖励表属于静态代码，不写入存档。
- 存档只保留进度、计数、确定性随机种子和最近结果。
- 最近结果最多保存 20 条，避免存档无限增长。
- 所有抽取使用保存的种子和计数器，不能依赖 `Math.random()`。
- 扣券、生成结果、更新保底、发放道具和保存最近记录必须在一个纯函数事务中完成。
- 未知卡片 ID、无效奖励和越界数字在 normalize 时过滤或钳制，不能导致整个存档无法加载。

## 8. 页面与交互

- “共同梦想”使用独立页面，不加入新的首页大型卡片。
- 页面顶部显示四项梦想总完成度与 Classic 最终目标。
- 中部显示四项梦想阶段、要求、已投入金币和投入按钮。
- 四项完成后该区域切换为无限纪念等级。
- 下方设置“回忆抽取”区域，显示回忆券数量、兑换入口、单抽、十连和保底进度。
- 十连结果使用固定网格，桌面为 5×2，390px 窄屏为 2×5，不能因名称长度改变布局。
- 抽取动画保持简短并允许跳过；结果必须在动画前已经结算。
- 卡册使用分类筛选和未获得占位，不显示限时或即将下架文案。
- 概率说明、保底规则和奖励池必须可以在抽取区域直接查看。

## 9. 受影响系统

- `PetState`、存档 normalize 与导入导出。
- 伙伴日程生涯分类计数与大师次数 selector。
- 道具类型、库存、物品图标和中英文文案。
- 新终局核心模块、抽卡核心模块和无副作用 selector。
- 首页入口、页面导航、终局页面和抽取结果弹层。
- 成就系统，可增加共同梦想完成、首次十连和卡册完成成就。
- 帮助页与许可证说明，不涉及现实货币或 DLC 授权。

## 10. 实施阶段与优先级

### P0：数据与核心规则

- 建立终局 schema v1、默认状态和迁移。
- 实现解锁判定、梦想阶段、分次投入和阶段完成。
- 实现回忆券兑换、单抽、十连、保底、重复卡和碎片兑换。
- 完成确定性随机与原子结算规则测试。

### P1：页面与文案

- 新增共同梦想页面、首页入口和导航。
- 完成梦想进度、投入确认、抽取、十连结果和卡册页面。
- 更新中英文 JSON、帮助页和相关提示。

### P2：平衡与收尾

- 根据满级玩家稳定收入确定 `C`、回忆券价格和各奖励权重。
- 检查卡池平均返还价值、120 天目标周期和无限纪念费用曲线。
- 补充最终纪念页、称号和 Classic 完成凭证。

## 11. 兼容性与风险

- 旧存档迁移时默认终局状态为空，不追溯自动完成梦想阶段。
- 既有伙伴日程累计计数可以用于阶段资格；缺失的旧计数只做保守推断，不能凭技能等级直接补满次数。
- 共同梦想投入不可撤销，必须防止重复点击、负数、溢出和并发结算。
- 十连必须先完整结算再展示，避免通过退出页面重抽。
- 抽卡奖励不能比金币兑换成本更高，否则会形成无限套利。
- 卡池内容永久固定；Classic 停更后不能依赖未来补池解决重复率或保底体验。
- 如果未来将抽卡与现实付费挂钩，必须重新评估地区法律、商店披露和未成年人规则；本计划明确不包含该变化。

## 12. 验证计划

- 验证 Lv.29、单项技能不足 Lv.6、全技能 Lv.6 三种解锁状态。
- 验证四项梦想所有阶段要求、分次投入、超额投入钳制和大额确认。
- 验证四项全部完成、最终纪念和无限纪念等级费用曲线。
- 验证金币兑换 1/10 张回忆券、余额不足和重复点击保护。
- 验证单抽、十连、十连珍贵保底、50 抽纪念保底及二者同时触发时的优先级。
- 验证确定性随机：同存档、同种子和同计数器得到完全一致的结果。
- 验证结果结算后关闭页面、重载或离线不会恢复回忆券或改变结果。
- 验证普通、珍贵、纪念奖励发放，以及重复卡碎片和定向兑换。
- 对固定种子运行大样本规则脚本，检查概率分布与平均金币返还比例。
- 验证 schema 缺字段、未知卡片、非法计数和旧存档迁移。
- 运行 JSON 解析、TypeScript 类型检查、核心规则脚本、`git diff --check` 和 `npm run build`。
- 静态检查桌面及 390px 窄屏下梦想进度、十连网格、概率说明和卡册布局，不执行截图式 Browser QA。

## 13. 当前假设

- Classic 终局以低维护、永久可用和离线单机为最高优先级。
- 抽卡只是可选的金币消耗与收藏玩法，不是共同梦想的必经随机门槛。
- 回忆券只能通过游戏内金币和少量固定奖励获得。
- 卡册复用现有图片，首版不生产新的大型 CG 资源。
- 具体金币整数价格和奖励权重必须在实现前通过当前经济数据重新计算。
