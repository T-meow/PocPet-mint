# 邻居氛围系统与多 Mod 库实施计划

## 目标

- 增加轻量、单机的邻居氛围，让邻居随机出现在伙伴日程、每日遭遇和离线事件中。
- 将单 Mod 槽位升级为最多 12 个完整 Mod 的本地资料库，未激活 Mod 的宠物名可作为邻居名。
- 保持现有伙伴日程收益、存档格式、Mod manifest 格式和未知 Mod 道具兼容行为。

## 范围与非目标

本轮包含多 Mod 安装、选择、删除与旧数据迁移；邻里日程叙事；具名邻居拜访、礼物和离线玩耍；中英文文案与 Mod 指南更新。

本轮不包含联网、真实玩家、好感度、独立社区页、多宠物存档、版本号调整或安装包生成。

## 实施阶段

1. 建立带版本号的 Mod 库目录，升级 IndexedDB 并迁移现有单 Mod 数据。
2. 增加邻居引用、日程 schema v4、确定性的每日 30% 邻里日程规则。
3. 为每日和离线事件传入当前邻居与可购道具上下文，实现分层礼物抽取。
4. 更新角色选择、设置管理、存档导入、双语文案和 Mod 制作指南。
5. 扩展核心规则检查，执行 JSON 解析、TypeScript/Vite 构建和差异检查。

## 兼容与数据风险

- 外部存档保持 schema v1，Mod manifest 保持 schema v1/v2；只升级内部伙伴日程到 schema v4。
- v3 及更旧的进行中/待领取日程不追溯增加邻居，下一个日程板开始参与新规则。
- 邻里日程只保存 Mod ID，显示时解析当前宠物名；Mod 删除或缺失时回退“邻居”泛称。
- Mod 导入在单个 IndexedDB 事务中替换同 ID 资源，配额或写入失败时保留原版本。
- 当前活动 Mod 的自定义道具可以成为礼物；切换 Mod 后继续沿用未知 Mod 道具占位行为。

## 礼物规则

- 礼物事件每次只发放 1 个道具，保留现有每日或离线事件的触发概率；每个 5:00 开始的游戏日最多结算 3 件邻居礼物，达到上限后改为其他事件；增益卡奖励未领取时为其保留 1 件。
- 99% 普通分支在当前可购道具中抽取价格不超过 1000 的道具。
- 1% 稀有分支中，金苹果占 60%；其余 40% 在价格超过 1000 的当前可购道具中抽取。
- 稀有分支没有高价候选时必定发放金苹果；普通池异常为空时回退苏打饼干。

## 验证计划

- 覆盖单 Mod 迁移、多 Mod 安装与更新、活动 Mod 切换/删除、数量上限和事务回滚。
- 覆盖邻居排除、名字更新与缺失回退、每日邻里日程稳定性和旧日程迁移。
- 用可控随机数覆盖普通/稀有礼物边界、价格边界、金苹果权重和空池回退。
- 运行伙伴日程规则脚本、双语 JSON 解析、`npm run build` 和 `git diff --check`。

## 假设

- Mod 库最多安装 12 个不同 Mod，同 ID 更新不占新名额。
- 内置 Furo 不作为具名邻居；没有未激活 Mod 时使用泛称。
- 不使用 Browser 截图式 QA，不执行打包。
