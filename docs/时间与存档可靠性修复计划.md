# 时间与存档可靠性修复计划

## 目标

- 防止系统时间回拨造成每日奖励重复、年度统计丢失和运行时计时冻结。
- 修复离线期间整段套用单一状态导致的属性、睡眠和健康结算跳变。
- 统一睡眠开始与唤醒路径，清理旧存档中的残留睡眠快照。
- 严格校验外部和内部存档，提供最近有效备份与可恢复的错误状态。
- 将每个好结局年份的长期奖励调整为每日金币 +30、额外爱心概率 +30%。

## 范围

### 包含

1. 主存档时间保护字段与旧存档迁移。
2. 每日额度、奖励和年度统计的单调日期策略。
3. 睡眠快照、手动唤醒、道具唤醒和自然醒。
4. 离线属性分段推进以及跨昼夜、伙伴日程、番茄钟边界。
5. 内部存档恢复、外部存档结构校验、导入确认和番茄钟剩余时间。
6. 好结局奖励效果和对应文案。

### 不包含

- 不改变宠物永不死亡的设计。
- 不增加联网校时、账号或云存档。
- 不试图完全阻止持续把系统时间向未来调整；本次重点是避免回拨循环、冻结和数据丢失。
- 不调整现有 Mod 格式和资源内容。

## 实施阶段

### 阶段一：时间保护

- 在 `PetState` 中增加带版本的时间保护状态，记录最后观察时间和最晚每日日期。
- 时间回拨时平移运行时期限，保留番茄钟、伙伴日程、花园、睡眠和冷却的剩余时长。
- 每日奖励与额度统一使用不倒退的日期键；天气、季节和界面日期继续使用真实系统日期。
- 超过 36 小时的明显回拨或滞留未来日期会把每日状态重映射到当前日并保留当日用量，同时修正不可能存在于未来的创建日期，避免长期锁定。
- 年度统计遇到未来年份时保留原数据，不再清空。

### 阶段二：睡眠与离线推进

- 新增统一的睡眠开始和唤醒入口。
- 清醒状态强制清空睡眠快照；睡眠状态的非法或未来快照安全重建。
- 将离线属性推进拆为有上限的小时间片，并在 05:00、07:00、22:00、闲置入睡、伙伴日程和番茄钟边界切段。
- 每段重新计算压力和衰减；离线随机事件、回归提示和睡醒奖励保持有界。

### 阶段三：存档可靠性

- 外部旧版裸存档必须满足稳定字段指纹，任意 JSON 对象不再被接受。
- 内部加载返回成功、缺失、可恢复或损坏状态；损坏数据不会自动写回。
- 保存最近有效备份，导入前备份当前进度；恢复和重置均由玩家明确选择。
- 导入先预览再确认，并按导出时刻保留运行中番茄钟的真实剩余时间。
- 导入替换采用可回滚的本地存储事务；确认期间重复点击、Mod 切换失败或主存档写入失败都不更新内存状态。

### 阶段四：好结局奖励

- 每个 `completedGoodEndingYears` 提供每日金币 +30。
- 每个完成年份提供额外爱心概率 +30%。
- 第一年的好结局 CG 保持不变；旧存档直接按已完成年份获得新效果。
- 删除不再使用的全局金币/爱心固定加成展示与结算路径。

## 兼容与数据风险

- 新字段全部提供默认迁移，旧存档无需手工转换。
- 时间保护的最晚日期从现有每日字段中推导，避免升级后重新开放已领取额度。
- 损坏主存档保留原文，并优先尝试最近有效备份。
- 离线多日只允许有限的睡醒副作用，避免修复后产生大量被动奖励。
- 外部 schema v1 继续读取；更高版本继续明确拒绝。

## 验证计划

- 新增时间与生命周期专项脚本：回拨、跨 22:00、跨 07:00、24 小时压力、伙伴日程、番茄钟和长离线性能。
- 新增存档恢复专项脚本：`{}`、截断 JSON、合法旧档、备份恢复、导入确认数据、睡眠基线、番茄钟剩余时间和写入失败回滚。
- 覆盖普通动作、三类道具和自然醒的睡眠快照清理。
- 验证好结局 0/1/2 年的每日金币与爱心概率累加。
- 回归现有日期奖励、花园、扭蛋、属性缩放、伙伴日程和生产构建。

## 已确认假设

- 宠物永不死亡是明确设计，不修改。
- 跨多晚离线可模拟状态变化，但一次离线恢复最多结算一次睡醒/梦境奖励。
- 好结局每年奖励采用每日金币 +30、额外爱心概率 +30%。

## 实施结果

- 2026-07-23：上述四个阶段均已完成。
- 时间、睡眠、离线生命周期、好结局与存档恢复专项脚本全部通过。
- 原有日期奖励、花园、扭蛋、属性缩放与伙伴日程回归脚本全部通过。
- `tsc --noEmit` 与 Vite 生产构建通过；仅保留既有的大分块体积提示。
