# 伙伴日程初版实施计划

## 目标

- 宠物达到 Lv3 后解锁“伙伴日程”，提供收益略高于连续打工、但需要现实时间完成的成长玩法。
- 每日生成 3 项确定性活动，统一使用单一现实时间倒计时，不与用户番茄钟共享进度。
- 活动期间保护宠物状态，并通过首页专用日程栏清楚呈现活动名称、进度和剩余时间。
- 保持种植园、商店、领奖、设置、存档、成就及用户番茄钟正常可用。

## 范围与非目标

本轮包含：

- Lv3 解锁；Lv8 起出现长活动。
- 每日 5:00 刷新的 3 项活动，以及短 45 分钟、标准 120 分钟、长 240 分钟三档时长。
- 四类熟练度、开始时固定状态消耗、活动完成后的奖励选择。
- schema v2 和 schema v1 活动迁移。
- 活动区间状态冻结、宠物照顾锁定、首页日程栏及相关中英文文案。

本轮不包含：

- 装饰系统、补给槽、活动 Mod、新货币、新素材和随机失败。
- 多日程并行、无限刷新、新成就和打包版本号调整。
- 浏览器截图式视觉 QA。

## 核心规则

- 每天按日期、宠物创建时间和等级确定性生成活动，同一天重复打开不会刷新结果。
- Lv3-Lv4 为 2 个短活动和 1 个标准活动；Lv5-Lv7 为 1 个短活动和 2 个标准活动；Lv8 起短、标准、长各一项。
- 一次只能进行一个活动；完成后必须领取收获，才能开始下一项。
- 活动统一按现实时间推进，关闭应用后继续倒计时。用户番茄钟可独立运行，但不会改变日程进度。
- 开始时一次扣除体力、饱腹和心情，并重置体力恢复基线；提前结束不返还消耗、不保留进度、不发放奖励。
- 活动覆盖的时间区间不计算饱腹、心情、清洁和健康自然变化，也不恢复体力；年龄、天气、花园和番茄钟继续推进。
- 活动结束后的离线时间恢复正常模拟；待领取状态不提供冻结。
- 活动进行时禁止玩耍、清洁、打工、睡觉、喂食、使用食物/礼物/护理品、触摸宠物和宠物升级。
- 背包浏览、购物、心心兑换、花园操作、领奖、设置、存档、成就、增益卡和用户番茄钟保持可用。

## 收益

- 收益基准沿用当前打工基础值：`24 + max(0, level - 1) + 秋季固定加成`。
- 金币分别为打工基准的 `1.6 × 1.15`、`4.2 × 1.15`、`8.5 × 1.15`。
- 短、标准、长活动的基础熟练度分别为 10、23、50。
- 标准和长活动可选择金币收获或分类收获；分类收获仍保留大部分金币。

## 存档与兼容

- `PartnerScheduleState.schemaVersion` 为 2。
- 进行中活动只保存 `startedAt`、`endsAt`、活动标识和奖励快照；待领取结果不保存模式。
- schema v1 的自主活动保留结束时间；一起活动按已完成专注比例折算为统一时长的剩余现实时间。
- 迁移保留活动、奖励快照、熟练度、已完成记录和待领取结果；到期活动在生命周期推进时立即转为待领取。
- 缺失字段使用受限默认值补齐，异常时间戳限制在单日最大范围内，避免时间回拨产生额外进度或重复奖励。

## 实施阶段

1. 升级伙伴日程类型、定义、收益、迁移、开始、推进、取消和领取规则。
2. 在宠物生命周期中按活动区间扣除自然模拟时间，并移除番茄钟推进接口。
3. 在核心操作入口增加统一的宠物忙碌判断，同时保留花园、购物、领奖和番茄钟路径。
4. 首页用专用日程栏替换照顾栏，调整日程页、背包、愿望、回归任务、宠物触摸和升级状态。
5. 更新中英文文案和响应式样式，执行规则检查、构建与差异校验。

## 验证计划

- 验证 Lv2 锁定、Lv3 解锁、Lv8 长活动和同日活动板稳定性。
- 验证 v1 自主、v1 一起、待领取和旧存档缺字段迁移。
- 验证短/标准/长活动在线、离线、跨日完成和重复领取保护。
- 验证活动区间状态完全冻结，结束后的离线时间正常衰减，待领取不冻结。
- 验证照顾、清洁、喂食、触摸、升级和快捷愿望在 UI 与 core 层均被阻止。
- 验证花园、购物、领奖、设置和用户番茄钟不受影响，番茄钟不会推进日程。
- 运行本地 core 规则检查、JSON 解析、`git diff --check` 和 `npm run build`。
- 人工检查代码中的桌面与 390px 窄屏约束，不执行截图式 Browser QA。

## 假设

- 用户番茄钟是独立生产力工具，允许与伙伴日程并行。
- 状态冻结只影响自然生命周期；番茄钟和奖励结算产生的金币、心情等奖励仍正常生效。
- 本轮继续使用现有 React/Tauri 架构，不新增依赖。
