# 增益卡重构与每日 5 点刷新实施计划

## 目标

- 好友证定位为低门槛的打工过渡增益，保持 10 心心价格，将打工加成调整为每次 8 金币、每日最多 80 金币。
- 挚友证价格调整为 90 心心，取消打工加成，改为伙伴日程金币加成 10%。
- 两种增益卡每日均可领取一份邻居礼物，并分别附带 15/50 金币。
- 将所有玩法日常边界统一为本地时间 5:00，同时保留生日、周年、节日、月初礼物和季节的自然日期边界。

## 范围与非目标

- 本轮包含增益卡数值、每日领取、邻居礼物上限、日程奖励、每日日期工具、存档兼容、中英文文案和核心规则检查。
- 本轮不改变伙伴日程 XP、分类奖励、大师次数、每日完成上限、增益卡有效期或现有邻居礼物池概率。
- 不补发错过日期的卡牌礼物，不退还历史挚友证购买差价，不调整生日等现实日期奖励。

## 实现规则

- 统一使用 `DAILY_RESET_HOUR = 5` 和游戏日日期键；04:59 属于前一游戏日，05:00 开始新游戏日。
- 好友证为每日金币 15、打工每次 +8/每日 80、额外心心 10%；挚友证为每日金币 50、日程金币 +10%、额外心心 30%，原种植增益不变。
- 同一游戏日两张卡共享一次领取资格。领取时从现有完整邻居礼物池发放一件道具，并附加当前生效卡的金币。
- 邻居礼物每日总上限保持 3。持卡且当日奖励未领取时，随机礼物最多结算 2 次，为卡牌礼物保留一个位置。
- 若购卡前已收到 3 件随机礼物，当日卡牌领取只发金币，次日 5:00 后恢复保证礼物。
- 挚友证日程金币加成与技能金币加成相加，在日程开始时写入奖励快照，不影响 XP、物品和大师次数。

## 兼容与数据风险

- `BoostCardState` 升级为 schema v2，使用共享的 `dailyRewardClaimed`，移除旧领取卡 ID 和未使用的额外心心日计数。
- 同一游戏日旧存档已有 `dailyCoinsClaimedCardId` 时迁移为已领取，避免更新后重复领取。
- 旧午夜日期键在 5:00 前映射到当前游戏日，保留已有次数和领取状态；旧 6:00 系统在 5:00 至 6:00 提前刷新属于预期变化。
- 已购买卡牌的到期时间和累计购买天数保持不变；挚友证新价格只影响后续购买。

## 验证计划

- 覆盖 04:59/05:00 边界、旧日期键迁移、卡牌 schema v1 迁移、切卡防双领和邻居礼物上限。
- 覆盖好友证 +8/80、挚友证价格 90、挚友证无打工加成及日程金币 +10% 的加法叠加和奖励快照。
- 覆盖两卡完整礼物池、Mod 道具、泛称邻居、空池回退和购卡前礼物已满场景。
- 运行核心规则脚本、双语 JSON 解析、`npm run build` 和 `git diff --check`。
