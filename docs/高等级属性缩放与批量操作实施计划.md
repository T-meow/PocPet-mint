# 高等级属性缩放与批量操作实施计划

## 目标

- 让饥饿、心情、清洁、健康的日常产出与消耗随属性上限同比增长，避免高等级后单次操作影响过小。
- 保持体力相关消耗、恢复、物品效果和阈值为固定数值，避免体力经济随等级放大。
- 20 级后开放商店与背包的批量操作，减少重复点击，同时保持交易和使用逻辑原子化。

## 范围与非目标

### 本次范围

- 四项属性使用 `当前属性上限 / 100` 作为动态倍率，覆盖动作、互动、事件、番茄钟、伙伴日程及连续操作反馈。
- 四项属性的状态判断由固定数值改为相对上限比例。
- 物品的正负效果保持固定单件数值；批量使用时按件数线性累计。
- 20 级后商店显示 1-99 数量选择器，背包显示 `x1 / x5 / x10` 快捷档位；购买和使用在 20 级前强制为 1。
- 批量购买共享统一报价逻辑，首件可享每日折扣，其余按原价；苏打饼干受每日剩余额度限制。
- 批量使用普通食物、礼物、护理品；金苹果、生日蛋糕和花园相关使用流程仍为单件。
- 数量选择在同一次弹窗会话内保留，关闭并重新打开弹窗后重置为 1。
- 抚摸互动的基础小心心按 `max(1, round(属性上限 / 100))` 增长，成就与增益卡奖励每次互动只结算一次。

### 非目标

- 不调整体力上限成长、自然恢复速度、动作体力消耗、物品体力效果或体力阈值。
- 不改变存档结构和 Mod 物品格式。
- 不重做商店或背包整体视觉设计。

## 实施阶段

1. 在属性核心层增加倍率与比例判断工具，并保留四项属性的小数精度，避免逐秒结算吞掉衰减。
2. 将非物品来源的四项属性变化接入动态倍率，体力逻辑保持原样。
3. 增加批量报价、批量购买和批量使用核心接口，保证库存、金币和每日额度不足时整批失败。
4. 为商店增加旧 WebView 兼容的数量步进器，背包使用同样兼容的 `x1 / x5 / x10` 分段按钮，并实现等级门槛与会话状态。
5. 增加聚焦检查脚本，运行全部现有检查、TypeScript/Vite 构建和 Android 测试包打包。

## 影响文件与系统

- `src/core/petStats.ts`：属性倍率、比例阈值、小数精度。
- `src/core/petActions.ts`、`petLifecycle.ts`、`petEvents.ts`、`petCommon.ts`：动作、时间和事件结算。
- `src/core/partnerSchedule.ts`、`pomodoro.ts`、`dailyWishes.ts`、`petState.ts`：外围产出、消耗与状态判断。
- `src/ui/ShopModal.tsx`、`InventoryModal.tsx`、`App.tsx`：批量交互与会话状态。
- `src/styles.css`、`src/styles/dialogs.css`：兼容旧 WebView 的数量控件布局。
- `src/i18n` 与检查脚本：动态文案和回归覆盖。

## 兼容性与数据风险

- 存档字段不变，现有整数属性可直接读取；新版本可能保存四项属性的小数值。
- UI 继续显示取整后的属性，内部保留小数只用于精确累计。
- Mod 物品效果仍按定义中的固定数值计算，不随等级缩放。
- 核心层再次校验等级、数量、库存、金币和每日限额，避免仅靠 UI 限制造成绕过。

## 验证计划

- 覆盖 1、20、50、99 级，确认四项属性倍率正确且体力变化完全不缩放。
- 比较逐秒推进与一次性长时间推进，确认自然衰减结果一致。
- 覆盖批量数量边界、等级门槛、库存不足、金币不足、每日折扣和饼干限额。
- 覆盖同弹窗内数量保留、使用后数量收缩及关闭重开后重置。
- 覆盖 1、20、50、99 级抚摸互动，确认基础小心心分别为 1、2、3、6，且心情消耗保持同比缩放。
- 覆盖普通内置物品和 Mod 物品，确认物品效果保持固定。
- 运行全部 `check:*`、`npm run build`，再按当前版本生成 Android arm64 debug 签名测试包。

## 假设

- 当前 `package.json` 的版本号是发布版本唯一来源；本次保留现有 `1.4.7`。
- “使用后不清零”指同一次背包弹窗会话中的数量选择不重置；重新打开弹窗时恢复为 1。
