# 冒险 DLC 存档兼容计划

本文档只记录未来冒险 DLC 的兼容设计，不代表当前版本已经实现冒险玩法，也不要求立即修改现有存档结构。

## 目标

- 冒险 DLC 作为同应用扩展，直接读取当前基础宠物存档。
- 当前 `PetState` 继续作为宠物身份、等级、属性、金币、小心心和背包的唯一来源。
- 冒险数据独立保存，避免破坏当前休闲养成玩法和旧版本导入能力。
- 当前存档、裸 `PetState` JSON、现有 `PocPetSaveFileV1` 都可以直接导入冒险模式。

## 当前存档事实

- 基础宠物本地存档键为 `pocpet.pet.v1`。
- 当前导出格式为 `PocPetSaveFileV1`，顶层包含 `schemaVersion: 1`、`app: "PocPet"`、`exportedAt`、`pet` 和可选 `activeMod`。
- 当前导入逻辑支持两类输入：带 `pet` 的 PocPet 存档包装对象，以及裸 `PetState` 对象。
- 当前 `PetState` 已包含等级、五项属性、金币、小心心、背包、天气、睡眠、番茄钟等长期状态。
- 现有 mod 系统主要用于替换宠物图、商品图和文本，不适合作为第一版冒险玩法运行时。

## 兼容原则

- 不把冒险进度直接塞进 `PetState`，避免基础养成逻辑和冒险逻辑互相污染。
- 冒险扩展使用可选字段 `extensions.adventure`；没有该字段时，冒险模式按当前宠物状态生成默认冒险档。
- 顶层导出格式暂时保持 `schemaVersion: 1`，新增字段只做向后兼容扩展；旧基础版导入时可以忽略未知字段并读取 `pet`。
- 新版导入时必须同时支持：
  - 当前 `localStorage` 的 `pocpet.pet.v1`。
  - 裸 `PetState` JSON。
  - 当前 `PocPetSaveFileV1`。
  - 未来包含 `extensions.adventure` 的完整导出文件。
- 基础宠物存档仍保存在 `pocpet.pet.v1`；冒险本地进度另存为 `pocpet.adventure.v1`。

## 建议导出结构

```json
{
  "schemaVersion": 1,
  "app": "PocPet",
  "exportedAt": "2026-06-25T00:00:00.000Z",
  "pet": {},
  "activeMod": {
    "id": "example",
    "name": "Example",
    "version": "1.0.0"
  },
  "extensions": {
    "adventure": {
      "schemaVersion": 1,
      "materials": {},
      "equipmentInventory": [],
      "equipped": {},
      "shopStockDate": "2026-06-25",
      "shopStock": [],
      "activeRun": null,
      "recentResults": []
    }
  }
}
```

说明：

- `pet` 仍是必需字段，冒险模式必须先成功恢复基础宠物。
- `extensions.adventure` 是可选字段，缺失时用默认冒险状态补齐。
- 旧版本基础应用重新导出存档时可能不会保留 `extensions`，这是可接受的降级行为。

## 冒险状态建议

`AdventureStateV1` 建议包含：

- `schemaVersion`：冒险扩展自己的版本号，第一版为 `1`。
- `materials`：冒险材料数量，例如矿石、布料、徽记等。
- `equipmentInventory`：装备背包，装备使用稳定字符串 ID。
- `equipped`：当前装备槽位，例如武器、护符、背包。
- `shopStockDate` 和 `shopStock`：每日冒险商店库存。
- `activeRun`：正在进行的派遣；没有派遣时为 `null`。
- `recentResults`：最近战报和结算摘要。

`AdventureRun` 建议包含：

- 地点 ID、开始时间、计划结束时间。
- 出发时的宠物等级、体力、健康、心情快照。
- 出发时的装备快照。
- 随机种子，用于保证结算结果稳定，避免刷新刷奖励。

## 数值继承

- 冒险战力由当前宠物状态临时计算，不写回基础等级公式。
- 建议公式：`pet.level * 8 + floor((energy + health + mood) / 6) + equipmentPower + weatherBonus`。
- 派遣限制：
  - 睡眠中不可出发。
  - 番茄钟运行中不可出发。
  - 健康 `<= 35` 不可出发。
  - 体力低于地点要求不可出发。
- 出发时消耗体力；结算时可轻微影响心情、清洁和健康。
- 小心心不进入冒险商店，继续只用于当前升级系统。
- 金币可购买基础冒险物品；高级装备和强化材料主要由冒险材料承担。

## 冒险玩法第一版

- 第一版采用“动画战报”，不实现实时或逐回合战斗循环。
- 玩家选择地点、时长和装备后开始派遣。
- 派遣中显示倒计时、地点、天气影响和简单动画。
- 到时后生成战报：遭遇怪物、过程描述、结果、奖励、状态变化。
- 可支持提前召回；召回只给少量或不给奖励，并减少宠物状态损耗。
- 地点建议先做三档：
  - 15 分钟：低难度，低体力消耗，主要给少量金币和基础材料。
  - 45 分钟：中难度，稳定材料收益，有小概率装备掉落。
  - 90 分钟：高难度，高体力要求，更高奖励和轻微健康风险。

## 天气影响

- 晴天：冒险战力 `+5`，成功后心情额外 `+1`。
- 多云：无额外影响，作为基准天气。
- 雨天：材料奖励 `+1`，但清洁额外 `-4`。
- 微风：派遣时长缩短 `10%`，提前召回惩罚减轻。

## UI 入口

- 冒险入口放在状态栏下方功能栏，和升级、番茄钟、天气并列或扩展为第二行。
- 冒险面板包含地点选择、装备、当前派遣、最近战报和冒险商店。
- 基础商店继续保留原入口；冒险商店只出现在冒险面板内，避免和当前道具商店混淆。

## 导入导出规则

- 导入旧基础存档：恢复 `pet`，创建默认冒险状态。
- 导入新冒险存档：恢复 `pet`，再恢复 `extensions.adventure`。
- 导入裸 `PetState`：按旧存档处理，创建默认冒险状态。
- 导入包含未知装备 ID 或未知材料的冒险存档：忽略未知项，保留可识别项，并给出提示。
- 导入存在进行中派遣的冒险存档：按当前时间结算；如果时间异常，则暂停该派遣并提示玩家重新进入冒险面板处理。

## 测试清单

- 旧 `pocpet.pet.v1` 本地存档升级后能直接进入冒险模式。
- 裸 `PetState` JSON 能导入，并生成默认冒险状态。
- 当前 `PocPetSaveFileV1` 能导入，并保留等级、金币、小心心、属性和背包。
- 带 `extensions.adventure` 的存档能恢复材料、装备、商店、进行中派遣和最近战报。
- 新冒险存档导入旧基础版时，旧版仍能读取 `pet`。
- 战力计算覆盖低等级、高等级、低体力、低健康和装备加成情况。
- 派遣限制覆盖睡眠、番茄钟运行、低健康、低体力。
- 战报结算由随机种子固定，同一派遣不会因刷新页面反复刷奖励。
- 冒险商店扣除金币和材料正确，小心心不被消耗。
- 移动窄屏下冒险入口、功能栏和冒险面板不重叠。

## 非目标

- 不引入联网真实天气。
- 不引入服务器存档。
- 不做真实平台 DLC 付费校验。
- 第一版不做实时战斗、复杂技能树或多人内容。
- 第一版不改变当前宠物升级消耗公式。
