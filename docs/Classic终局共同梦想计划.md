# Classic 终局：共同梦想计划

## 1. 目标

- 在 Classic 停止大型内容更新前，提供一个能长期消耗金币、收束现有养成系统并拥有明确结局的终局玩法。
- 让宠物等级、伙伴日程技能、大师次数、花园和日常积累在终局阶段继续有价值。
- 为长期玩家提供明确且可完成的四项共同梦想，以及完成主线后的永久金币出口。
- 控制新美术和内容生产成本，优先复用现有宠物活动图片、道具图标和 UI 组件。

共同梦想与纪念抽卡是两个独立系统。本计划不包含回忆券、随机卡池、抽卡保底、重复卡或回忆碎片；纪念抽卡由独立计划实现，双方互不作为解锁前置条件。

## 2. 范围与非目标

### 本轮范围

- 新增“共同梦想”终局页面与四项长期工程。
- 新增梦想阶段投入、阶段结算、最终纪念和无限纪念等级。
- 新增独立的终局存档 schema、旧存档迁移、中英文文案和核心规则检查。
- 新增称号、共同梦想纪念记录和 Classic 完成凭证。

### 非目标

- 不实现回忆券、抽卡、卡池、保底、碎片兑换或卡册。
- 不加入排行榜、联网校验、交易、玩家间赠送或云端结算。
- 不加入自由摆放装饰、冒险地图、实时战斗或新的大型资源生产系统。
- 不改变当前代码中的宠物等级上限，也不追溯修改现有养成进度。

## 3. 金币基准

共同梦想使用固定设计基准 `C = 2,000` 金币。

- `C` 以 Lv.30 终局玩家拥有完整增益、成熟花园和招财树等高位产出时的综合收益为参照，并取整为 2,000 金币。
- `C` 只是设计期换算单位。发布版本必须保存下文给出的明确整数价格，不能在运行时根据玩家等级、余额、季节、增益或实际收入动态变化。
- 招财树单次 jackpot、库存补给带来的重复打工、番茄钟累计时长等非稳定极值不继续放大 `C`。
- 即使未来经济产出发生调整，本版共同梦想价格也保持不变；如需重新平衡，必须通过显式版本变更处理。

Lv.30 是本计划的 Classic 终局平衡锚点，不代表本轮要把代码中的 `maxPetLevel` 改为 30。

## 4. 解锁条件

- 宠物达到 Lv.30。
- 四项伙伴日程技能全部达到 Lv.6。
- 解锁后首页功能区显示“共同梦想”入口。
- 未解锁时可以查看条件进度，但不能投入金币或完成梦想阶段。

解锁条件的目的，是让终局建立在现有养成完成度上，而不是成为新玩家早期的金币陷阱。

## 5. 四项共同梦想

四项梦想与伙伴日程技能一一对应：

| 分类 | 梦想 | 最终成果 |
|---|---|---|
| 学习 | 伙伴百科 | 完成一本记录共同生活的百科 |
| 料理 | 四季食谱 | 整理一册四季料理记录 |
| 园艺 | 纪念果园 | 建成一座属于伙伴的纪念果园 |
| 运动 | 漫长环行 | 完成一次象征成长的漫长环行 |

### 5.1 阶段结构

每项梦想包含五个固定阶段：

| 阶段 | 技能要求 | 分类日程要求 | 大师要求 | 金币投入 |
|---|---:|---:|---:|---:|
| 1 | Lv.6 | 累计完成 10 次 | 无 | 2,000 |
| 2 | Lv.7 | 累计完成 25 次 | 无 | 4,000 |
| 3 | Lv.8 | 累计完成 50 次 | 无 | 8,000 |
| 4 | Lv.10 | 累计完成 80 次 | 10 次 | 16,000 |
| 5 | Lv.10 | 累计完成 120 次 | 30 次 | 30,000 |

- 每项梦想总投入为 60,000 金币。
- 四项梦想全部完成总投入为 240,000 金币。
- 旧玩家可以用既有积蓄加速，但仍需满足技能和生涯次数要求。
- 分类日程要求读取成就计数器中的生涯完成次数，避免重复维护相同统计。

### 5.2 投入规则

- 金币可以分次投入，进度永久保留且不能撤回。
- 提供“投入 10%”“投入 25%”“补足当前阶段”三个操作。
- 百分比按当前阶段总价计算并向下取整，最低有效投入为 1 金币。
- 实际投入不能超过当前阶段剩余需求，也不能让金币余额变为负数。
- 补足金额超过设定的大额阈值时需要二次确认。
- 技能或次数尚未达标时仍可提前投入金币，但只有金币、技能、日程次数和大师次数全部满足后才能完成阶段。
- 完成阶段后展示一段简短成长记录，不增加新的现实时间等待。

### 5.3 阶段与最终奖励

- 阶段奖励以纪念和收藏为主，不返还大量金币。
- 每个阶段给予少量心心、普通道具或独立的共同梦想纪念记录。
- 每项梦想完成后解锁对应称号和固定成果记录；该记录不属于随机卡池，也不依赖纪念抽卡状态。
- 四项梦想全部完成后解锁 Classic 最终纪念页、最终称号和完成凭证。
- 最终纪念文案明确表示 Classic 主线完成，但宠物仍可继续照顾和养成。

### 5.4 无限纪念等级

四项梦想全部完成后解锁“共同纪念等级”：

- 第 1 级需要投入 20,000 金币。
- 后续每级需求为上一级的 1.35 倍，使用四舍五入后的整数金币。
- 金币可以分次投入；未完成当前等级前不能向后续等级投入。
- 只记录纪念等级、生涯累计投入和每 10 级称号。
- 不提供金币、属性或生产效率加成，防止终局经济再次膨胀。
- 该系统作为 Classic 停更后的永久金币出口，不设置等级上限。

## 6. 数据结构与迁移

新增独立的 `ClassicEndgameState`，初始 schema 为 v1：

```ts
interface ClassicEndgameState {
  schemaVersion: 1;
  projects: Record<PartnerScheduleCategory, DreamProjectProgress>;
  completedAt?: number;
  legacyLevel: number;
  legacyCoinsInvested: number;
}
```

- `ClassicEndgameState` 不包含 `MemoryDrawState`、回忆券、卡册或任何抽卡字段。
- 梦想定义、阶段价格、阶段文案和奖励表属于静态代码，不写入存档。
- 存档只保留各阶段投入、阶段完成状态、最终完成时间、无限纪念等级和累计投入。
- 旧存档迁移时默认终局状态为空，不追溯自动完成梦想阶段或补发奖励。
- 未知分类、越界阶段、负数投入和非法时间戳在 normalize 时过滤或钳制，不能导致整个存档无法加载。
- 扣除金币、增加投入、完成阶段和发放奖励必须在一次纯函数事务中结算，避免重复点击产生部分成功状态。

## 7. 页面与交互

- “共同梦想”使用独立页面，不加入新的首页大型卡片。
- 页面顶部显示四项梦想总完成度与 Classic 最终目标。
- 中部显示四项梦想的当前阶段、资格要求、已投入金币、剩余金币和投入操作。
- 四项完成后，梦想投入区域切换为无限纪念等级。
- 页面不显示回忆券数量、抽卡入口、保底进度或卡册入口。
- 阶段完成和最终纪念动画保持简短并允许跳过；结果必须在动画前完成结算。
- 桌面与窄屏布局均需保证阶段价格、要求和操作按钮不会重叠或改变卡片尺寸。

## 8. 受影响系统

- `PetState`、存档 normalize 与导入导出。
- 伙伴日程生涯分类计数与大师次数 selector。
- 新终局核心模块和无副作用 selector。
- 首页入口、页面导航、共同梦想页面和最终纪念页。
- 成就系统，可增加单项梦想完成、四项梦想完成和纪念等级里程碑成就。
- 中英文文案与帮助页。

纪念抽卡模块、回忆券和卡册不属于本计划的实现依赖。

## 9. 实施阶段与优先级

共同梦想在独立纪念抽卡完成后实施。

### P0：数据与核心规则

- 建立终局 schema v1、默认状态和旧存档迁移。
- 实现解锁判定、五阶段规则、分次投入、原子结算和最终完成状态。
- 实现无限纪念等级价格曲线和累计投入。

### P1：页面与文案

- 新增共同梦想页面、首页入口和导航。
- 完成条件进度、投入确认、阶段结果、最终纪念和无限等级界面。
- 更新中英文 JSON、帮助页和相关提示。

### P2：奖励与收尾

- 配置阶段普通奖励、称号、固定成果记录和 Classic 完成凭证。
- 检查 60,000/240,000 金币总价和无限纪念费用曲线。
- 完成兼容、边界和布局验证。

## 10. 兼容性与风险

- 当前代码等级上限可能高于 Lv.30；共同梦想只读取 `level >= 30`，不能借本功能修改或钳制现有等级。
- 既有伙伴日程累计计数可以用于阶段资格；缺失的旧计数只做保守推断，不能凭技能等级直接补满次数。
- 共同梦想投入不可撤销，必须防止重复点击、负数、溢出和并发结算。
- `C = 2,000` 和所有阶段整数价格必须使用静态配置，不能因未来经济变化自动漂移。
- 无限纪念等级无上限，费用计算和累计投入必须使用现有安全整数钳制规则。
- 共同梦想的固定成果不能占用纪念抽卡卡池 ID，也不能要求玩家先拥有回忆券或卡册。

## 11. 验证计划

- 验证 Lv.29、Lv.30、单项技能不足 Lv.6、全技能 Lv.6 等解锁状态。
- 验证五阶段价格分别为 2,000、4,000、8,000、16,000、30,000，每项合计 60,000，四项合计 240,000。
- 验证四项梦想所有技能、分类日程和大师次数要求。
- 验证分次投入、百分比取整、补足、余额不足、超额投入钳制和大额确认。
- 验证资格未达标时允许预先投入，但不能提前完成阶段或领取奖励。
- 验证四项全部完成、最终纪念、完成凭证和无限纪念等级解锁。
- 验证无限纪念第 1 级为 20,000，后续按 1.35 倍四舍五入且累计投入正确。
- 验证重复点击、结算中断、导出导入和重载不会重复扣款或重复发奖。
- 验证 schema 缺字段、未知分类、非法阶段、负数投入和旧存档迁移。
- 运行 JSON 解析、TypeScript 类型检查、核心规则脚本、`git diff --check` 和 `npm run build`。
- 静态检查桌面及 390px 窄屏下的进度、价格、要求和投入操作，不执行截图式 Browser QA。

## 12. 当前假设

- Classic 终局以低维护、永久可用和离线单机为最高优先级。
- Lv.30 是终局功能的解锁与经济锚点，不是对现有等级上限的修改。
- `C = 2,000` 是人工确认的固定设计值，不追踪玩家实时最大收益。
- 共同梦想不依赖随机抽取，所有玩家都能通过确定性投入完成主线。
- 共同梦想成果使用独立纪念记录，首版不生产新的大型 CG 资源。
